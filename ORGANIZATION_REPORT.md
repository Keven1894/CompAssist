## Recommendations for Maintenance

### When Adding New Features
1. Add code to appropriate `src/` module
2. Update `src/README.md` with new module info
3. Update main `README.md` if adding CLI command
4. Add documentation to relevant `docs/` subdirectory
5. Update subdirectory README
6. Update `INDEX.md` with new content
7. Update `.cursor/rules.md` if needed

### When Adding New Documentation
1. Place in appropriate `docs/` subdirectory
2. Update subdirectory's README.md
3. Update `docs/README.md`
4. Update `INDEX.md`
5. Add cross-references from related docs

### When Refactoring
1. Update source code
2. Update `src/README.md`
3. Update any affected documentation
4. Update `PROJECT_SUMMARY.md` if major changes
5. Verify all links still valid

### Regular Maintenance
- Review documentation quarterly
- Verify all links work
- Update version numbers
- Check for outdated information
- Sync with code changes
